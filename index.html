What is SDLC?

A structured process for building software that ensures quality, efficiency, and maintainability.

📝 Planning

Requirements gathering, feasibility study, project charter & scope definition

📐 Design

System architecture, database schema, API contracts, wireframes & prototypes

💻 Development

Writing code, code reviews, unit tests, documentation — the actual building phase

🧪 Testing

QA testing, integration tests, user acceptance testing, performance benchmarks

🚀 Deployment

CI/CD pipelines, release management, blue-green deployments, canary releases

📊 Monitoring

Logs, alerts, performance metrics, dashboards, incident response

🔁 Maintenance

Bug fixes, feature updates, tech debt reduction, patches & new versions

02

SDLC Models

🏔️

Waterfall

Sequential, phase-by-phase approach. Requirements locked upfront. Best for fixed-scope projects with clear requirements.

✓ Clear milestones ✗ No flexibility
// Flow:
Plan → Design → Build → Test → Deploy
// Each phase completes before next starts
🔄

Agile / Scrum

Iterative sprints (2-week cycles). Deliver working software each sprint. Adapt to changing requirements.

✓ Flexible ✓ Fast feedback
Sprint 1: MVP ─→ Sprint 2: Auth
Sprint 3: Payments ─→ Sprint 4: Launch 🚀
// Continuous improvement
♾️

DevOps

Continuous integration & delivery. Automate everything. Ship to production multiple times per day.

✓ Fast delivery ⚡ Complex setup
Code → Build → Test → Deploy → Monitor
  ↑_________________________________↓
// Infinite loop of improvement

Agile Ceremonies

The rituals that keep agile teams aligned and productive

🗓️

Sprint Planning

Team decides what to build this sprint. Product owner prioritizes backlog. Team estimates effort using story points.

Every 2 weeks
📅

Daily Standup

15-minute sync: What I did yesterday, what I'll do today, any blockers. Keep it short and actionable.

Every day, 15 min
🔍

Sprint Review

Demo completed work to stakeholders. Get feedback. Adjust product backlog based on insights.

End of sprint
🔄

Retrospective

What went well? What to improve? Action items for next sprint. Continuous team improvement.

End of sprint

05

Pull Request Workflow

1
Create Branch & Push

git checkout -b feature/user-auth && git push origin feature/user-auth

2
Open Pull Request

Write a description, link to issue, add reviewers. GitHub runs CI/CD checks automatically.

3
Code Review

Teammates review code, leave comments, request changes. Iterate until everyone approves.

4
CI Checks Pass ✅

All automated tests, linting, security scans pass. Branch is up to date with main.

5
Squash & Merge 🎉

Merge to main. Delete feature branch. Deploy to production automatically via CI/CD.

The CAP Theorem

In distributed systems, you can only guarantee 2 of 3 properties:

🔄

Consistency

All nodes see the same data at the same time

✅

Availability

Every request gets a response (even if it's stale)

🌐

Partition Tolerance

System works even if network splits

CI/CD with GitHub Actions

📝 Push / PR Trigger
→
🔨 Build Compile + Docker
→
🧪 Test Unit + Integration
→
🔒 Security SAST + Deps
→
🚀 Deploy Staging → Prod
# .github/workflows/ci.yml
name: CI/CD Pipeline
on:
  push: { branches: [main, develop] }
  pull_request: { branches: [main] }

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npm run build && npm test

  deploy-production:
    needs: build-and-test
    if: github.ref == 'refs/heads/main'
    environment: production # Manual approval gate

Introduction to Generative AI

The technology reshaping how we build, write, and create software

What is Generative AI?

AI systems that can create new content — code, text, images, audio — by learning patterns from massive datasets. Unlike traditional AI that classifies or predicts, Gen AI generates.

// Traditional AI:
Input: "Is this email spam?" → Output: Yes/No

// Generative AI:
Input: "Write an email to..." → Output: Full email
Input: "Build a REST API for..." → Output: Working code

Key Foundation Models

🧠 Large Language Models (LLMs)

GPT-4, Claude, Gemini, LLaMA — trained on text, generate text & code

🎨 Diffusion Models

DALL-E, Midjourney, Stable Diffusion — generate images from text

🔀 Multimodal Models

GPT-4o, Gemini — understand text, images, audio, video together
